Navegação

Navegação global

A barra de navegação global é formada por várias partes. A barra em si é um .nav-global-container, dentro de um .nav-global-inner, dentro de um .nav-global. Adicionando um .nav-global-fixed-top a barra se fixa ao topo da tela. Quando a barra esta fixa no topo, é preciso que o body receba um padding-top de pelo menos 42px, já que a barra irá sempre encobrir o topo.

Os botões da esquerda são links .nav-global-button dentro de um .nav-global-buttons. O botão que estiver ativo no momento deve ter também .nav-global-button-active.

Os botões da direita são botões com dropdown. Os botões são de estilo padrão (.button-default) com .dropdown-toggle, icon-xxx-lightblue_16_18-before e atributo data-toggle="dropdown". Ao ser clicado, o .dropdown-menu é mostrado.

Cada dropdown tem suas particularidades que são bem específicas a aplicação. Somente dando uma olhada na sua estrutura é que ficará claro a sua compreenão.

Breadcrumbs

Breadcrumb (mini)

Este breadcrumb é formado por links .breadcrumb-mini-link, dentro de .breadcrumb-mini-item, dentro de .breadcrumb-mini. Para remover o texto do breadcrumb, deixando somente o ícone, basta adicionar .breadcrumb-mini-compact. Cada item do breadcrumb, com exceção do último, deve utilizar o ícone do separador .icon-arrow-right-separator-gray_16_18-after. Cada link deve utilizar o ícone icon-xxx-lightblue_16_18-before.

Breadcrumb (ícones)

Cada item deste breadcrumb é um .breadcrumb-icon, dentro de um .breadcrum-icon-link (caso exista um link), dentro de um .breadcrumb-item, dentro de um .breadcrumb-icons. O item que estiver ativo no momento deve receber .breadcrumb-icon-active.

Dependendo do estado do item, o ícone deve usar uma classe diferente. Se o item não foi alcançado ainda, ele deve usar .icon-xxx-lightgray_32_34-before para indicar um estado de desabilitado. Se o item está ativo no momento, ele deve usar .icon-xxx-lightblue_32_34-before. Se o item é anterior ao item ativo, ele deve usar .icon-xxx-gray_32-34-before.

Tabs e sub-tabs

Estas abas são englobadas por um bloco de classe .tabs. Cada aba deve ficar dentro de uma classe .tab. A aba que estiver ativa no momento deve ter também a classe .tab-active.

O título da aba é um link com a classe .tab-title que deve ter também a classe do ícone (.icon-xxx-lightblue_16_18-before). Existe também um elemento de classe .tab-sub-title com um texto que só é exibido pela aba que está ativa.

Os botões abaixo das abas devem estar englobados por um bloco com a classe .tab-buttons. Cada botão deve ter a classe .tab-button e a classe do ícone (.icon-xxx-lightblue_16_18-before). O botão que estiver ativo no momento deve ter também a classe .tab-button-active e a classe do ícone deve ser branca (.icon-xxx-white_blue_16_18-before).

Cabeçalhos

Este cabeçalho é formado por .header-item dentro de .header. O link deve ter .header-link e a classe do ícone cinza, exceto no último item que deve ser azul claro. Outra exceção é quando uma página de aula está ativa, neste caso o último item da lista deve ter .header-item-subject e o ícone utilizado deve ser cinza (ver o último exemplo).

Quando alguma página de AVA, curso ou disciplina estiver ativa, o cabeçalho .header-breadcrumb-big deve ser usado. Este elemento funciona da mesma maneira do cabeçalho de início, mas usa uma fonte maior.

Quando uma página de aula de aula está ativa, já se deve usar outros elementos que são mais específicos.

Filtros (visão geral)

Os filtros da visão geral são bem específicos. Eles são formados por .filter dentro de .filter-group e .filters-general-view.

Os filtros com menu dropdown devem ter a classe .dropdown-toggle e o atributo data-toggle="dropdown". Isso é necessário para que o JavaScript do dropdown funcione. Quando o filtro for clicado, o elemento .dropdown-menu é mostrado.

Outras classes como: .filter-everything usado para identificar o filtro que deve exibir tudo; .filter-courses usado para identificar o filtro de cursos, pois este exibe um comportamento diferente do sub-título quando os cursos e disciplinas são selecionados.

A estrutura dos checkboxes funciona como uma lista aninhada. O primeiro nível da lista deve ter a classe .filter-level-1 e seus itens filhos a classe .filter-level-1-item. Os níveis interiores seguem o mesmo padrão, lista com .filter-level-2 e itens com .filter-level-2-item e por assim vai.

Filtros

Para criar estes elementos utilize a classe .filter dentro de um bloco com a classe .filters. Toda vez que um filtro é clicado, ele alterna entre o estado de ativado e desativado. Para mostrar um filtro no estado ativado, basta adicionar a classe .filter-active. Apenas um filtro pode permanecer ativado.

<div class="filters">
  <!-- FOR EACH FILTER -->
  <button class="filter [filter-active] [icon-xxx-lightblue_16_18-before]">FILTER_TEXT</button>
  <!-- END -->
</div>
